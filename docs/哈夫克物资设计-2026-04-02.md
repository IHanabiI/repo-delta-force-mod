# 哈夫克物资设计（2026-04-02）

本文档用于固定当前第一版哈夫克事件与哈夫克物资池的统一设计口径。

## 1. 总体流程

第一版的固定开局流程如下：

1. 玩家进入一局游戏。
2. 开局约 5 秒后，`哈夫克事件` 百分百触发。
3. HUD 显示提示：
   `哈夫克公司投放了特殊物资，收集它们以获得额外收益。`
4. 当前地图引入 `哈夫克物资池`。
5. 第一版哈夫克物资池中包含 3 种物资：
   - 军用信息终端
   - 飞行记录仪
   - 航空箱

## 2. 命名统一

当前项目中：

- `黑匣子` 与 `飞行记录仪` 视为同一个物资。
- 后续策划、文档、代码口径默认优先使用 `飞行记录仪`。
- 如果需要兼顾早期概念描述，可以写成：
  `飞行记录仪（黑匣子）`

## 3. 三种物资的定位

### 3.1 军用信息终端

定位：

- 情报型工具物资
- 作用重点是帮助玩家发现高价值目标
- 不承担恐怖氛围核心职责

设计关键词：

- 工具
- 情报
- 搜集辅助
- 有限次数追踪
- 六格电量槽

### 3.2 飞行记录仪（黑匣子）

定位：

- 哈夫克事件中的高价值、高风险、异常氛围物资
- 不是雷达工具，而是恐怖感与不稳定性的来源

当前确认的设计口径：

- 只有 `手持时生效`
- 当前碎裂阈值已经足够脆，不需要再额外调得更脆
- 生效时制造异常环境效果
- 破碎后效果立刻停止
- 价值会随着损坏快速下跌，直到归零

异常效果的当前优先级理解：

- `门异常`
- `屏幕/残响/环境扰动`
- `怪物失向`

这几类效果的优先级目前大致相近，不需要强行把门异常或怪物失向排到屏幕/残响之前。

补充说明：

- `怪物失向` 目前实现难度较高，可作为后续补充目标
- 当前版本优先保证“手持后产生明显异常氛围、摔碎后立即结束”这一主链路稳定成立

设计关键词：

- 高价值
- 易碎
- 手持触发
- 异常环境
- 恐怖氛围

### 3.3 航空箱

定位：

- 特殊容器型物资
- 核心玩法是“完整回收”与“提前开启”之间的收益取舍

当前确认的设计口径：

- `完整状态` 时是高收益物资
- `提前开启` 后会降为低收益
- 开启后获得的 `即时收益` 本身也只是低收益
- 因此开启航空箱本质上是用较小的即时收益，换掉完整回收时的大收益

物理/外观性格定义：

- `偏大`
- `偏重`
- `偏中等坚硬`

这意味着它在体感上应区别于普通小件物资：

- 不应显得轻巧
- 不应像飞行记录仪一样脆
- 但也不是极端耐久的重装甲物件

设计关键词：

- 容器
- 取舍
- 完整高收益
- 开启低收益
- 偏大偏重偏中等坚硬

## 4. 第一版设计重点

第一版 3 个物资的分工是：

- 军用信息终端负责 `找`
- 飞行记录仪负责 `险`
- 航空箱负责 `取舍`

这三者不应互相抢定位：

- 军用信息终端不是恐怖物件
- 飞行记录仪不是搜集工具
- 航空箱不是异常环境物件

## 5. 当前默认口径

如果后续没有新的明确修改，当前线程默认按以下口径继续推进：

- 哈夫克事件开局约 5 秒百分百触发
- 事件提示文案为“哈夫克公司投放了特殊物资，收集它们以获得额外收益”
- 第一版哈夫克物资池固定包含军用信息终端、飞行记录仪、航空箱
- 黑匣子默认等同于飞行记录仪
- 军用信息终端保留工具定位，不改回 valuable
- 军用信息终端可放入物品栏并继续后台扫描
- 军用信息终端只有手持时显示完整 HUD
- 军用信息终端底部六格 HUD 正式纳入设计，作为电量槽使用
- 军用信息终端第一次锁定目标免费，之后每次切换到下一个目标消耗 3 格电量
- 军用信息终端电量耗尽后显示“电量不足”，并隐藏部分目标引导 HUD
- 飞行记录仪只有手持时生效
- 飞行记录仪当前价值口径固定为 `9998`
- 飞行记录仪保持高价值、易碎、不能放入物品栏的方向
- 航空箱完整回收高收益，开启后与即时收益都为低收益

## 6. 当前实装进度补充（2026-04-03）

当前阶段已经明确的实装状态如下：

- 军用信息终端与飞行记录仪都已经完成基础接入，并能在游戏中生成、拿取、测试
- 飞行记录仪当前方向已经基本稳定：
  - 只有手持时生效
  - 碎裂阈值已经足够脆
  - 红光表现已满足当前需求
  - 不能进入物品栏
  - 价值口径固定为 `9998`
- 军用信息终端当前仍保留官方工具类底层链路，因此手持时会出现底部六格 HUD
- 这组六格 HUD 长时间尝试移除未果，现已正式转为设计资源，不再当作必须删除的 bug
- 当前下一阶段重心是：
  - 先把飞行记录仪与军用信息终端收尾
  - 后续再正式开始航空箱制作

当前仍需记住的发布前清理项：

- 目前为了测试，开局强制投放 / 定向生成通道曾被临时打开
- 第一版正式收尾时，需要把这些纯测试用行为恢复为正常逻辑
- 用于测试的飞行记录仪 HUD 等临时行为也需要一起清理
